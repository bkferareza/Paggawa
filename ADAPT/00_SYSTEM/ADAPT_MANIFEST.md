## What ADAPT Means in This Workspace

ADAPT is the delivery operating system for Paggawa.

It turns product source truth into bounded implementation work.

ADAPT must preserve:

- product intent
- decisions
- plans
- validation criteria
- build history
- scope boundaries
- current execution state

This workspace is not a maintenance workspace and not a migration workspace.

This workspace is for creating a new product prototype from a defined product baseline.

---

## Current Objective

The first objective is to create a working prototype of Paggawa.

Paggawa v0.1 is modeled as two app surfaces under one shared product system:

- Paggawa Mobile for resident and worker phone/mobile use
- Paggawa Quest for barangay hall assisted-access workflows

The surfaces share one workspace, domain model, mock/local data layer, validation model, and ADAPT delivery system.

The prototype must demonstrate:

- resident job posting
- worker discovery
- nearby job browsing
- barangay-assisted access
- quest board visibility
- basic matching
- simple completion and reputation tracking
- privacy before match

The first goal is not a production-scale platform.

The first goal is a usable and explainable prototype.
