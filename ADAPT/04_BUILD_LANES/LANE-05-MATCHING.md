# LANE-05 - Matching

STATUS: NOT_STARTED

## Purpose

Allow workers to respond to jobs and residents to accept a worker in Paggawa Mobile.

## Required Inputs

- Job Requests lane
- Worker Discovery lane
- Quest Board lane

## Intended Output

A basic match can be created between a job request and a worker in the shared data layer.

## In Scope

- Paggawa Mobile worker response
- Paggawa Mobile resident shortlisting/acceptance
- match status
- contact/address privacy rule

## Out of Scope

- full chat
- payments
- escrow
- real-time notifications
- complex negotiation flow

## Validation Criteria

- Worker can respond to an open job in Paggawa Mobile.
- Resident can view worker response.
- Resident can accept worker.
- Job status changes to matched.
- Exact address/contact is not exposed before match.
