# MVP Acceptance Checklist

The first Paggawa prototype is acceptable if the following are true:

- [ ] Paggawa Mobile Resident can post a job.
- [ ] Paggawa Mobile Worker can see nearby jobs.
- [ ] Paggawa Mobile Resident can see nearby workers.
- [ ] Paggawa Quest Barangay Staff can create a job for a resident.
- [ ] Paggawa Quest Barangay Staff can register or view a worker.
- [ ] Paggawa Quest quest board shows open jobs.
- [ ] Worker can respond to a job.
- [ ] Resident can accept a worker.
- [ ] Exact address/contact is not exposed before match.
- [ ] Completed job can update worker reputation.
- [ ] Paggawa Mobile and Paggawa Quest share the same mock/local data layer.
